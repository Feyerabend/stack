
## Direct Execution vs Loader and Linkers

In the folder [01](./01/) you will find an example that shows how to go directly from assembly
code to an executable, where the execution is performed by a RISC-V emulator. This demonstrates
the most direct path: hand-written assembly is assembled and immediately run in the emulator
without any intermediate build stages. Naturally, the assembly could also be the output from a compiler.

In the folder [02](./02/) you can see the full toolchain pipeline, including the intermediate
steps between assembly (for example, assembly code generated by a compiler) and the final executable,
which is again executed using an emulator (lightly modified). This folder illustrates how object 
iles are produced, how they are linked together, and how the final binary is created before execution.

To support this workflow, the assembler has been modified (obj_asm.py) so that its output is
suitable for use by a linker. In particular, it now produces *relocatable code* instead of
*fixed-address binaries*. Additionally, some minor changes have been made to the emulator
to correctly handle the resulting executable format. Together, these modifications make it
possible to demonstrate a more realistic compilation and linking process similar to what
happens in a real toolchain.
