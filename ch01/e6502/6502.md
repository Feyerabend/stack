
## MOS 6502 Assembly Reference



### Programming model

The 6502 is an 8-bit CPU with a 16-bit address bus (64 KB flat address space).
It has a minimal register set; most work flows through the accumulator.

#### Registers

| Register | Bits | Description                                                         |
|----------|------|---------------------------------------------------------------------|
| *A*      | 8    | Accumulator — all arithmetic and logic                              |
| *X*      | 8    | Index register; offset in indexed modes                             |
| *Y*      | 8    | Index register; offset in indexed modes                             |
| *SP*     | 8    | Stack pointer — points into page $01 ($0100–$01FF), full-descending |
| *PC*     | 16   | Program counter                                                     |
| *P*      | 8    | Processor status (flags)                                            |

#### Status flags (P register)

```
  7   6   5   4   3   2   1   0
  N   V   —   B   D   I   Z   C
```

| Bit | Flag | Name      | Set when                                      |
|-----|------|-----------|-----------------------------------------------|
| 7   | *N*  | Negative  | Result bit 7 = 1                              |
| 6   | *V*  | Overflow  | Signed arithmetic overflow                    |
| 5   | —    | Unused    | Always 1                                      |
| 4   | *B*  | Break     | BRK instruction executed (in stack copy only) |
| 3   | *D*  | Decimal   | BCD mode active (not implemented on NES)      |
| 2   | *I*  | Interrupt | Maskable interrupts disabled                  |
| 1   | *Z*  | Zero      | Result = 0                                    |
| 0   | *C*  | Carry     | Carry out / borrow                            |



### Memory layout

```
$0000 – $00FF    Zero page                — fast 2-byte instructions; indirect pointers live here
$0100 – $01FF    Stack                    — hardware stack (SP = $FF on reset, grows down)
$0200 – $FFFF    General RAM / ROM / I/O  — system-defined
$FFFA – $FFFB    NMI vector
$FFFC – $FFFD    RESET vector             — CPU reads this on power-on/reset
$FFFE – $FFFF    IRQ/BRK vector
```



### Addressing modes

`n` = bytes, `c` = cycles.  "+1" = one extra cycle if operand address crosses a
page boundary.  "+2" = two extra on page cross (read-modify-write).

| Mode               | Syntax        | n | c     | Notes                           |
|--------------------|---------------|---|-------|---------------------------------|
| *Implied*          | `NOP`         | 1 | 2     | No operand                      |
| *Accumulator*      | `ASL A`       | 1 | 2     | Operates on A                   |
| *Immediate*        | `LDA #$42`    | 2 | 2     | Literal value                   |
| *Zero page*        | `LDA $80`     | 2 | 3     | Address $00–$FF                 |
| *Zero page, X*     | `LDA $80,X`   | 2 | 4     | (`$80`+X) & $FF                 |
| *Zero page, Y*     | `LDX $80,Y`   | 2 | 4     | LDX/STX only                    |
| *Absolute*         | `LDA $1234`   | 3 | 4     | Full 16-bit address             |
| *Absolute, X*      | `LDA $1234,X` | 3 | 4+1   | +2 for RMW                      |
| *Absolute, Y*      | `LDA $1234,Y` | 3 | 4+1   | STA is always 5                 |
| *Indirect*         | `JMP ($1234)` | 3 | 5     | JMP only; 6502 page-wrap bug    |
| *Indexed indirect* | `LDA ($40,X)` | 2 | 6     | Ptr at ZP($40+X)                |
| *Indirect indexed* | `LDA ($40),Y` | 2 | 5+1   | Ptr at ZP$40, +Y                |
| *Relative*         | `BEQ label`   | 2 | 2+1+1 | Branch: +1 taken, +1 page cross |



### Instruction set

#### Load and store

| Mnem  | Operation | Modes | Flags |
|-------|-----------|---------------------------------------------|-----|
| *LDA* | A ← M     | imm zp zp,X abs abs,X abs,Y (ind,X) (ind),Y | N Z |
| *LDX* | X ← M     | imm zp zp,Y abs abs,Y                       | N Z |
| *LDY* | Y ← M     | imm zp zp,X abs abs,X                       | N Z |
| *STA* | M ← A     | zp zp,X abs abs,X abs,Y (ind,X) (ind),Y     | —   |
| *STX* | M ← X     | zp zp,Y abs                                 | —   |
| *STY* | M ← Y     | zp zp,X abs                                 | —   |

#### Register transfers

All are implied mode, 2 cycles, affect N Z (except TXS).

| Mnem  | Operation | Flags |
|-------|-----------|-------|
| *TAX* | X ← A     | N Z   |
| *TAY* | Y ← A     | N Z   |
| *TXA* | A ← X     | N Z   |
| *TYA* | A ← Y     | N Z   |
| *TSX* | X ← SP    | N Z   |
| *TXS* | SP ← X    | —     |

#### Stack (page $01)

| Mnem  | Operation | Cycles | Flags |
|-------|-----------|--------|-------|
| *PHA* | push A    | 3      | —     |
| *PHP* | push P    | 3      | —     |
| *PLA* | pull A    | 4      | N Z   |
| *PLP* | pull P    | 4      | all   |

#### Arithmetic

ADC and SBC use the carry flag as an extra input bit.  Always `CLC` before the
first ADC in a chain; `SEC` before the first SBC.

| Mnem  | Operation         | Modes                                       | Flags   |
|-------|-------------------|---------------------------------------------|---------|
| *ADC* | A ← A + M + C     | imm zp zp,X abs abs,X abs,Y (ind,X) (ind),Y | N V Z C |
| *SBC* | A ← A − M − (1−C) | imm zp zp,X abs abs,X abs,Y (ind,X) (ind),Y | N V Z C |

##### Increment / decrement

| Mnem  | Operation | Modes             | Cycles  | Flags |
|-------|-----------|-------------------|---------|-------|
| *INC* | M ← M + 1 | zp zp,X abs abs,X | 5 6 6 7 | N Z   |
| *DEC* | M ← M − 1 | zp zp,X abs abs,X | 5 6 6 7 | N Z   |
| *INX* | X ← X + 1 | impl              | 2       | N Z   |
| *INY* | Y ← Y + 1 | impl              | 2       | N Z   |
| *DEX* | X ← X − 1 | impl              | 2       | N Z   |
| *DEY* | Y ← Y − 1 | impl              | 2       | N Z   |

#### Logic

| Mnem  | Operation | Modes | Flags |
|-------|-------------------|---------------------------------------------|-------|
| *AND* | A ← A & M         | imm zp zp,X abs abs,X abs,Y (ind,X) (ind),Y | N Z   |
| *ORA* | A ← A \| M        | imm zp zp,X abs abs,X abs,Y (ind,X) (ind),Y | N Z   |
| *EOR* | A ← A ^ M         | imm zp zp,X abs abs,X abs,Y (ind,X) (ind),Y | N Z   |
| *BIT* | N←M7, V←M6, Z←A&M | zp abs                                      | N V Z |

#### Shift and rotate

All accept accumulator (A) or memory (zp, zp,X, abs, abs,X).
Memory forms: zp=5, zp,X=6, abs=6, abs,X=7 cycles.

```
      ASL    C ← [7][6][5][4][3][2][1][0] ← 0    shift left, bit 0 = 0
      LSR    0 → [7][6][5][4][3][2][1][0] → C    shift right, bit 7 = 0
      ROL    C ← [7][6][5][4][3][2][1][0] ← C    rotate left through carry
      ROR    C → [7][6][5][4][3][2][1][0] → C    rotate right through carry
```

| Mnem  | Flags                     |
|-------|---------------------------|
| *ASL* | N Z C                     |
| *LSR* | N Z C  (N always cleared) |
| *ROL* | N Z C                     |
| *ROR* | N Z C                     |

#### Compare

Comparison subtracts without writing back.  Sets flags as if `SBC` ran, but A/X/Y are unchanged.

| Mnem  | Operation | Modes                                       | Flags |
|-------|-----------|---------------------------------------------|-------|
| *CMP* | A − M     | imm zp zp,X abs abs,X abs,Y (ind,X) (ind),Y | N Z C |
| *CPX* | X − M     | imm zp abs                                  | N Z C |
| *CPY* | Y − M     | imm zp abs                                  | N Z C |

C is set (≥), Z is set (=), N is set (bit 7 of result = 1).

#### Branches

All are relative mode, signed 8-bit offset, 2/3/4 cycles.

| Mnem  | Branch if       | Condition |
|-------|-----------------|-----------|
| *BCC* | carry clear     | C = 0     |
| *BCS* | carry set       | C = 1     |
| *BEQ* | equal / zero    | Z = 1     |
| *BNE* | not equal       | Z = 0     |
| *BMI* | minus           | N = 1     |
| *BPL* | plus            | N = 0     |
| *BVC* | overflow clear  | V = 0     |
| *BVS* | overflow set    | V = 1     |

Range: −128 to +127 bytes from the instruction after the branch.
For longer jumps use `BXX skip / JMP target / skip:`.

#### Jump and call

| Mnem  | Mode | Cycles | Operation                             |
|-------|------|--------|---------------------------------------|
| *JMP* | abs  | 3      | PC ← addr                             |
| *JMP* | ind  | 5      | PC ← [addr] (see page-wrap bug below) |
| *JSR* | abs  | 6      | push PC−1, PC ← addr                  |
| *RTS* |      | 6      | pull PC, PC ← PC+1                    |
| *RTI* |      | 6      | pull P, pull PC                       |
| *BRK* |      | 7      | push PC+1, push P (B=1), PC ← [$FFFE] |

#### Flag instructions

All implied, 2 cycles.

| Mnem  | Effect               |
|-------|----------------------|
| *CLC* | C ← 0                |
| *SEC* | C ← 1                |
| *CLD* | D ← 0                |
| *SED* | D ← 1                |
| *CLI* | I ← 0  (enable IRQ)  |
| *SEI* | I ← 1  (disable IRQ) |
| *CLV* | V ← 0                |

#### Miscellaneous

| Mnem  | Cycles | Effect       |
|-------|--------|--------------|
| *NOP* | 2      | No operation |



### Interrupt system

#### Vectors

| Vector | Address | Triggered by                                                      |
|--------|---------|-------------------------------------------------------------------|
| RESET  | $FFFC   | Power-on or hardware RESET pin                                    |
| IRQ    | $FFFE   | Hardware IRQ (maskable, I flag must be 0)                         |
| NMI    | $FFFA   | Non-maskable interrupt (cannot be disabled)                       |
| BRK    | $FFFE   | Software interrupt (shares vector with IRQ; B flag distinguishes) |

#### Interrupt sequence (IRQ / NMI / BRK)

1. Complete current instruction.
2. Push PCH, PCL, P onto the stack.
3. Set I flag (mask further IRQs).
4. Load PC from the appropriate vector (little-endian).

RTI restores P (including I) then PC.

#### RESET sequence

1. Set I flag.
2. Read PCH:PCL from $FFFC:$FFFD.
3. Jump to that address.  SP and registers are undefined after reset.

#### BRK vs. IRQ

Both use the $FFFE vector.  The handler distinguishes them by checking the *B*
flag in the copy of P on the stack: B=1 means BRK, B=0 means hardware IRQ.

### Common patterns and idioms

#### 16-bit addition

```asm
; result16 = lo16 + hi16  (all in zero page)
CLC
LDA lo16
ADC hi16
STA result16
LDA lo16+1
ADC hi16+1
STA result16+1
```

#### 16-bit increment

```asm
INC ptr
BNE skip
INC ptr+1
skip:
```

#### Copy A to X (no TAX-free path needed, but shows the pattern)

```asm
TAX             ; 2 cycles
; or
STA tmp         ; 3 cycles (zero page) — useful if X must be preserved
LDX tmp
```

#### Unsigned multiply (8-bit × 8-bit → 16-bit, shift-and-add)

```asm
; A × multiplicand → result (2 bytes)
; Destroys: A, X, multiplicand
    LDX  #8
    LSR  A              ; shift multiplier into carry loop
mul_lp:
    BCC  mul_no
    CLC
    ADC  multiplicand
mul_no:
    ROR  A
    ROR  result+1
    ROR  result
    DEX
    BNE  mul_lp
```

#### Unsigned divide (16-bit ÷ 8-bit, shift-and-subtract)

Standard shift-and-subtract; see `basic/basic.asm` for a 16-bit ÷ 16-bit version.

#### Test bit n of A

```asm
AND  #%00001000     ; test bit 3; Z=1 if clear
```

#### Branch on A = 0

```asm
TAX                 ; sets Z without changing A
BEQ  is_zero
```

Or:

```asm
ORA  #0             ; sets N and Z; no other effect
BEQ  is_zero
```

#### Negate A (two's complement)

```asm
EOR  #$FF
CLC
ADC  #1
```

Or:

```asm
SEC
SBC  tmp            ; tmp = 0: result = 0 - A = -A
```

#### Delay loop (approximate)

```asm
; ~(5 × N) cycles  (LDX=2, DEX=2, BNE=3 taken / 2 not taken)
LDX  #N
delay:  DEX
        BNE delay
```



### Known hardware quirks

#### JMP indirect page-wrap bug

`JMP ($10FF)` reads the low byte from $10FF and the high byte from *$1000*
(not $1100).  The address never crosses a page boundary.  Avoid placing
indirect JMP vectors at the last byte of a page.

#### Decimal mode (D flag)

ADC and SBC interpret operands as BCD when D=1.  The N, V, Z flags are
unreliable in decimal mode.  The NES variant of the 6502 (Ricoh 2A03) omits
decimal mode entirely.

#### Unofficial / undocumented opcodes

The 6502 has 105 unused opcode bytes.  Many perform combinations of two
official operations (e.g., LAX = LDA+TAX, SAX = A&X→mem).  They work on
real hardware but differ between NMOS/CMOS variants.  This assembler
assembles only the 56 documented mnemonics.
