
## §5.3 — Types as Ontology

Companion notes for §5.3 of *The Language Stack*. Where the rest of this chapter
treats types operationally — symbol tables, the λ-calculus, inference, ownership
— §5.3 steps back to the *ontological* question the book raises: a type system is
a claim about what the values of a program **are**, and what distinctions are
worth making about them. These essays are reading material, not code.

| File | What it covers |
|------|----------------|
| [`language/PHILOSOPHY.md`](./language/PHILOSOPHY.md) | Philosophical questions on language — meaning, reference, and what a type "says" about a value |
| [`language/LINGUISTICS.md`](./language/LINGUISTICS.md) | Linguistics and computational linguistics: grammar, categories, and the parallels with type systems |
| [`reference/COMPILERS.md`](./reference/COMPILERS.md) | Annotated references on compilers |
| [`reference/INTERPRETERS.md`](./reference/INTERPRETERS.md) | Annotated references on interpreters |

The `language/` essays argue the conceptual side (what types mean); the
`reference/` files collect pointers for going deeper into the two ways a language
is realised — compiled and interpreted — a distinction the book returns to in
Chapters 6 (interpreter) and 9 (code generation).
